# Contributing to eSentinel Lite

Thank you for your interest in contributing to eSentinel Lite!

## Code of Conduct

- Be respectful and constructive
- Focus on security and simplicity
- Test your changes thoroughly
- Document your modifications

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your fork
7. Create a Pull Request

## Testing Requirements

- Test on fresh Linux installation
- Verify no IP leaks
- Check systemd integration
- Validate nftables syntax
- Test with real VPN connection

## Code Style

- Python: Follow PEP 8
- Bash: Use shellcheck
- Comments: Clear and concise
- Security: Fail safely

## Security Reports

For security issues, please email security@opsec.ee directly.
Do NOT open public issues for security vulnerabilities.
